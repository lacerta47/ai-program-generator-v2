# Product

## Register

product

## Users

- 1차: 초등학교 저학년(7~10세) 학생. 읽기 속도가 느리고, 작은 글자·복잡한 UI에 쉽게 막힌다. 마우스 정밀도가 낮아 큰 탭타깃이 필요하다.
- 2차: 교사/보호자(게시판 운영·관리자), 일반 사용자.
- 사용 맥락: 교실 또는 집에서 "내가 만들고 싶은 프로그램"을 한국어 계획서로 적고, AI가 만들어준 결과물을 즉시 가지고 논다.

## Product Purpose

계획서(이름·모습·사용법·동작·기타)를 쓰면 Gemini가 HTML/CSS/JS 프로그램을 생성한다. 미리보기 → 수정 요청 → ZIP 다운로드 → 게시판 공유까지 한 흐름. 성공 = 아이가 어른 도움 없이 첫 프로그램을 만들고 게시판에 올리는 것.

## Brand Personality

깔끔한, 귀여운, 다정한. 장난감 같은 즐거움이 있지만 유치하지 않고 단정하다. 화면이 아이에게 말을 거는 듯한 친근한 한국어 마이크로카피.

## Anti-references

- 이전 목업(회색 카드 + 파란 기본 버튼)의 밋밋한 generic SaaS 룩.
- 이모지를 장식으로 도배하는 화면. 생동감은 CSS 모션과 컬러로 만든다.
- 브랜드 마스코트 없음. 단 예외 하나: 로딩 화면 한정의 "코딩하는 로봇" CSS 일러스트(BuilderBot, 2026-06-11 사용자 요청로 추가 — 외부 에셋 없이 직접 그린 오리지널).
- 정보 밀도 높은 어른용 대시보드 룩(Linear/Notion 그대로 이식 금지).

## Design Principles

1. **큰 것이 친절한 것** — 큰 글자(본문 17~18px), 큰 버튼(≥48px), 큰 여백. 아이의 눈과 손에 맞춘다.
2. **모션은 반응이다** — 누르면 눌리고, 성공하면 톡 튀어오른다. 상태 피드백·로딩·빈 화면에만 모션을 쓰고 장식성 안무는 금지. reduced-motion 항상 준수.
3. **한 화면 한 가지 일** — 아이가 다음에 뭘 해야 할지 한 눈에 보이게. 선택지는 적게, 라벨은 쉬운 말로.
4. **일관된 부품** — 같은 버튼·입력·칩이 모든 화면에서 같은 모양. components/ui 프리미티브만 사용.
5. **결과물이 주인공** — 생성된 프로그램 미리보기가 가장 크고 밝게. 앱 UI는 그것을 받치는 무대.

## Accessibility & Inclusion

- 본문 대비 ≥4.5:1, 큰 텍스트 ≥3:1. 라이트/다크 모두.
- 모든 모션에 prefers-reduced-motion 대안.
- 키보드 포커스 링 뚜렷하게, 탭타깃 최소 44×44px.
- 쉬운 한국어(저학년 어휘), 한 문장 안내.
